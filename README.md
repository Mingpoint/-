## <a name="1">mqtt协议</a>
### 特点：
	1.使用的发布/订阅消息模式，它提供了一对多消息分发，以实现与应用程序的解耦。
	2.对负载内容屏蔽的消息传输机制。
	3.对传输消息有三种服务质量（QoS）：
		1.最多一次，这一级别会发生消息丢失或重复，消息发布依赖于底层TCP/IP网络。即：<=1
		2.至多一次，这一级别会确保消息到达，但消息可能会重复。即：>=1
		3.只有一次，确保消息只有一次到达。即：＝1。在一些要求比较严格的计费系统中，可以使用此级别
	4.数据传输和协议交换的最小化（协议头部只有2字节），以减少网络流量
	5.通知机制，异常中断时通知传输双方	
### 身份：
	1.发布者
	2.代理(服务器)
	3.订阅者
